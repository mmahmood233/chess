# Chess вЂ” Real-Time Multiplayer Chess App

A full-stack multiplayer chess application built with a **Flutter** frontend and a **NestJS** backend. Players can find random opponents through a waiting room, invite friends directly by Player ID, and play real-time chess with full rule enforcement.

## Features

| Feature | Detail |
|---|---|
| Real-time multiplayer | WebSocket (Socket.io) вЂ” moves and events delivered instantly |
| Public matchmaking | Waiting room auto-pairs two players |
| Friend invitations | Challenge any player by their UUID |
| Server-side move validation | chess.js rejects every illegal move |
| Client-side legal-move hints | chess.dart highlights valid squares before you tap |
| White plays first | Enforced by server; `yourTurn` sent to white on game start |
| Turn notifications | In-game snackbar + animated "Your Turn" badge |
| Full end-state detection | Checkmate, stalemate, draw, threefold repetition, insufficient material |
| Game-over dialog | Colour-coded result shown to both players simultaneously |
| Persistent player identity | UUID stored in SharedPreferences, survives app restarts |
| Reconnection support | Server ignores stale socket disconnects; client re-registers on every reconnect |
| UUID game sessions | Every game gets a unique ID generated by PostgreSQL |

## Architecture

### Backend

```
AppModule
в”њв”Ђв”Ђ GameModule
в”‚   в”њв”Ђв”Ђ GameGateway   вЂ” WebSocket events (register, makeMove, leaveGame вЂ¦)
в”‚   в””в”Ђв”Ђ GameService   вЂ” chess.js validation, Prisma reads/writes
в””в”Ђв”Ђ WaitingRoomModule
    в”њв”Ђв”Ђ WaitingRoomController  вЂ” REST endpoints
    в””в”Ђв”Ђ WaitingRoomService     вЂ” matchmaking, invite flow, game creation
```

- **Authoritative server**: all move validation and game logic live here; the client never mutates game state directly.
- **Socket rooms**: each game UUID is a Socket.io room вЂ” broadcasts go only to the two players in that game.
- **Race-condition guard**: `handleDisconnect` checks whether the disconnecting socket is still the *active* socket for that player before cleaning up, preventing a reconnect from wiping the player's registration.

### Frontend

```
main.dart          вЂ” async init (SharedPreferences UUID), ProviderScope
providers/
  game_provider    вЂ” GameStateNotifier: WebSocket events в†’ state transitions + navigation
services/
  websocket_service вЂ” Socket.io wrapper with Completer-based connection awaiting
  api_service       вЂ” Dio HTTP client for REST endpoints
models/
  game_state        вЂ” Immutable GameState with _Undefined-sentinel copyWith
screens/
  main_menu_screen   вЂ” Play Online / Play a Friend / Player ID card
  waiting_room_screen вЂ” Animated matchmaking UI
  game_board_screen   вЂ” Board + player panels + game-over/resign dialogs
widgets/
  custom_chess_board  вЂ” Tap-to-move board, SVG pieces, legal-move hints
```

- **Single navigation source**: only `GameStateNotifier._handleGameStarted` navigates to the game board.
- **Lazy provider init avoided**: `MainMenuScreen` watches `gameStateProvider` so the WebSocket connects and registers the player immediately on app launch вЂ” ensuring invites can be received even before the player opens the waiting room.

## WebSocket Events

### Client в†’ Server
| Event | Payload | Purpose |
|---|---|---|
| `register` | `{ playerId }` | Bind socket to player UUID |
| `joinGame` | `{ gameId, playerId }` | Join Socket.io room for a game |
| `makeMove` | `{ gameId, playerId, move }` | Submit a move |
| `leaveGame` | `{ gameId, playerId }` | Resign / leave mid-game |

### Server в†’ Client
| Event | Payload | Purpose |
|---|---|---|
| `registered` | `{ playerId }` | Confirms registration |
| `gameStarted` | `{ gameId, whitePlayerId, blackPlayerId, fen, currentTurn }` | Game created or reconnect sync |
| `yourTurn` | `{ gameId }` | It is this player's turn |
| `moveMade` | `{ playerId, move, fen, pgn }` | A move was applied |
| `gameOver` | `{ winner, endReason, message }` | Game ended |
| `moveError` | `{ error }` | Move rejected |
| `opponentLeft` | `{ gameId, message }` | Opponent disconnected/resigned |
| `inviteReceived` | `{ inviterId, invitedId }` | Friend challenge incoming |
| `inviteDeclined` | `{ invitedId }` | Invited player declined |

## Testing Checklist

To exercise all features manually:

1. Start the backend: `npm run start:dev` in `backend/`
2. Open two browser tabs running the Flutter web app
3. **Public game**: click *Play Online* on both tabs вЂ” they match automatically
4. **Friend invite**: copy Player ID from Tab B в†’ Tab A clicks *Play a Friend* в†’ paste ID в†’ Send
5. **Illegal moves**: try moving a pawn 3 squares, moving a rook diagonally вЂ” moves are rejected
6. **Turn order**: white always moves first; after each move the other player's panel lights up
7. **Game over**: play to checkmate вЂ” result dialog appears on both sides simultaneously
8. **Resign**: tap the flag icon mid-game вЂ” opponent wins
